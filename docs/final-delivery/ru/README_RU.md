## Ключевой принцип системы

AIS реализован как система с "толстой базой данных":

```text
Браузер
  -> PHP thin API
    -> SQL Server stored procedures / triggers
      -> данные, проверки, бизнес-правила, аудит
```

PHP-слой не должен содержать бизнес-расчётов. Он отвечает за транспорт, сессии, валидацию запроса, вызов хранимой процедуры и возврат JSON. Основная логика принадлежит SQL Server.

## Состояние пакета V2

Поставочный комплект является поставочным комплектом программного продукта:

- включены PHP, JS, CSS, SQL, локальные vendor-библиотеки QR, runtime-заглушки и документация;
- исключены репозитории разработки, локальные зависимости сборки, черновые тестовые материалы, временные отчёты проверки и внутренние рабочие документы;
- PHP syntax check пройден для 69 PHP-файлов;
- секреты окружения должны передаваться через переменные окружения, а не храниться в коде.

## Важное замечание по развёртыванию

Код использует абсолютные web-пути под `/ais-system-ru/`. Для финального запуска рекомендуется разворачивать содержимое поставочного каталога приложения как активное содержимое каталога:

```text
C:\xampp\htdocs\ais-system-ru
```

Запуск как `/ais-system-ru` возможен только после согласованной проверки базовых URL и клиентских путей.
