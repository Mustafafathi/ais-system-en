# База данных и хранимые процедуры AIS V2

## 1. Общий принцип

SQL Server является центральным владельцем бизнес-логики AIS. PHP не формирует произвольный SQL из пользовательского ввода, а вызывает хранимые процедуры. Это обеспечивает:

- единый источник бизнес-правил;
- защиту от обхода логики через альтернативные UI;
- централизованный аудит;
- предсказуемые контракты для API;
- возможность менять правила без переписывания всех страниц.

Основная база: `Улучшенная`.  
Основная схема: `dbo`.

## 2. SQL-файлы поставки

| Файл | Назначение |
| --- | --- |
| `Database/Улучшенная.sql` | Базовая схема: таблицы, основные процедуры, триггеры, ключевые индексы и базовая логика. |
| `Database/Дополнения.sql` | Дополнительные процедуры для дашбордов, SKUD, обоснований, расписания и справочников. |
| `Database/Integration SP.sql` | Процедуры CSV-интеграции. |
| `Database/admin_control_plane.sql` | Административный центр, разделы интерфейса, доступы ролей, инфраструктурные операции. |
| `Database/admin_roles_permissions.sql` | Управление ролями и разрешениями. |
| `Database/admin_group_report_fix.sql` | Исправление/расширение отчёта по группе. |
| `Database/schedule_overview_migration.sql` | Расширенный обзор расписания, аудитории, корпуса, автосоздание занятий. |
| `Database/scheduled_reports_agent.sql` | Плановые отчёты, запуски, артефакты и доставки. |
| `Database/password_reset_database_mail.sql` | Восстановление пароля через Database Mail. |
| `Database/fix_trigger_notifications.sql` | Исправления уведомлений/триггеров. |

## 6. Правила изменения базы данных

1. Любое новое бизнес-правило оформлять как хранимую процедуру или изменение существующей процедуры.
2. PHP не должен обходить процедуры прямым SQL.
3. Перед миграцией делать backup.
4. Сначала применять на staging.
5. Для destructive-изменений готовить rollback.
6. Имена процедур, используемых UI как `action`, менять только с одновременным обновлением UI.
7. Триггеры проверять транзакционными тестами.
8. Все интеграционные изменения документировать в `INTEGRATIONS_RU.md`.
